Data Integration

A study on key contributions of the Clio project, a joint project between the IBM Almaden Research Center and the University of Toronto begun in 1999. Clio’s goal is to simplify information integration, by providing tools that help in automating and managing one challenging piece of the problem which is conversion of data between representations. Clio mainly described about the schema mapping, specifications between data in two heterogeneous schemas. Clio provides algorithms to address the needs of two major information integration problems, namely, data integration and data exchange

Cloud Computing

In this research project,the concept of EC2 compute units is used to denote the compute capacity in GFlops of each instance. Each VCPU of the instance is a hyper thread of the Intel Xeon processor. considering eight computing instances m3.large.private, m3.2xlarge.private, c3.8xlarge.private, g2.2xlarge.private, r3.4xlarge.private, i2.8xlarge.private, d2.8xlarge.private and m4.10xlarge.private which are compared to the amazon ec2 instances m3.large, m2.2xlarge c3.8xlarge, g2.2xlarge, r3.4xlarge, i2.8xlarge,d2.8xlarge and m4.10xlarge.

The cost per hour of private cloud instances for each amazon instances that are compared . To build a private cloud the factors considered are Processor, Storage, Memory, Motherboard, Network card, Network Switch, Cooling Power, System Power, Chassis, Rack, UPS and Administration cost.

Software System Architecture

Designed two different Account components using a Model-Driven Architecture (MDA). Aspects that vary between the two Account components include Maximum number of times incorrect pin can be entered, Minimum Balance, Display Menu etc. MDA EFSM captures the generic behaviour of both the Account Components components and is decoupled from the data and implementation details. Three design patterns were used to implements the design which were state pattern, strategy pattern and abstract factory pattern. State patterns helps in the transition of states while the strategy pattern that enables the algorithm behaviour to be selected at run-time. Abstract Factory pattern provides an interface to create a family of related objects.
